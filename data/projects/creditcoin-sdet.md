# Gluwa – Distributed Validation Infrastructure

## 🔭 Overview

At Gluwa Inc, I worked as a Software Engineer on distributed systems and validation infrastructure for the
Creditcoin blockchain ecosystem — a multi-chain credit network spanning Ethereum, BNB Chain, Bitcoin, and a
custom Substrate-based L2. My work covered the full spectrum from CI/CD-integrated validation systems to
production cross-chain bridge workers.

## 🛠️ Core Work

### ⛓️ Multi-Chain Distributed Validation Infrastructure
- 🏗️ Designed distributed validation infrastructure for multi-chain blockchain systems (Ethereum, BNB Chain,
  Bitcoin, Substrate), ensuring transactional consistency and state correctness across networks
- 🔧 Built scalable backend automation services in C# (.NET 6) and Node.js to validate smart contract execution,
  REST APIs, and on-chain data pipelines
- 📐 Implemented SDK-level validation frameworks using BDD and OOP principles for reliable blockchain integrations
- 🤝 Collaborated with backend engineers on API architecture, performance validation, and system design for
  Web3 investment infrastructure

### 🔄 CI/CD-Integrated Test Orchestration
- 🏛️ Architected CI/CD-integrated test orchestration systems supporting DApps and protocol upgrades
- 🐳 Contributed to reliability engineering across distributed blockchain environments with Docker-based deployments
- 📡 Maintained automated pipelines across testnet, devnet, and mainnet environments

### 🌉 Cross-Chain Off-Chain Bridge Worker (CCNext)
- 📡 Built production NestJS off-chain worker monitoring Ethereum Sepolia for ERC-20 transfer events
- 🔁 Implemented exponential backoff proof API orchestration (20 retry attempts, 15-min timeout) for reliable
  cross-chain proof generation and submission to CCNext L2
- 🔢 Per-wallet nonce management for parallel transaction execution across multiple wallets
- ☁️ Full multi-environment deployment via Docker Compose and Azure App Configuration

### 🧪 Substrate Pallet Test Framework (CC3 Protocol)
- 🧪 Designed comprehensive Polkadot.js + Mocha + Allure test framework for Substrate pallet validation
- 📋 Coverage: staking lifecycle (bond, nominate, unbond, payout), Grandpa migration, precompile testing
  (BN128 curves, EVM signature verifier), and dynamic extrinsic calling
- 🌐 Multi-environment support: testnet, devnet, dryrun
- 📊 Structured Allure test reporting for team observability

### 🎭 E2E Automation (Playwright)
- 🎭 Implemented Playwright E2E test suites for Creditcoin staking dashboard and CC3 blockchain dashboard
- 🔄 Regression coverage for DeFi front-ends with CI integration

### 📜 Smart Contract Testing & Deployment
- 🔐 Authored Foundry-based Solidity tests for EIP-712 airdrop contract — cryptographic claim flows,
  campaign lifecycle, role-based access control
- 💱 Deployed and validated GluwaDEX (Uniswap V3 fork: SwapPool V1/V2, SwapRouter) with full test coverage
- 🏭 Uniswap V3 protocol deployment: factory, pool, router, NFT position manager

### 🔍 QA Mini-Indexer (On-Chain/Off-Chain Validator)
- 📡 Built a real-time dual-scanner validation tool monitoring on-chain events (WebSocket + HTTP polling fallback)
- ✅ Cross-validated on-chain state against off-chain indexer APIs for continuous production health monitoring
- 📋 Covered: airdrop claims (RewardsAdded, Claimed events), staking contracts (Staked, RewardClaimed, Unstaked)

## ⭐ Key Highlights
- 🏗️ 4+ years of production blockchain infrastructure work across 4 blockchain ecosystems simultaneously
- 🌉 Cross-chain bridge worker handling proof API orchestration with full retry resilience
- 🧪 Custom Substrate pallet test framework — testing a novel blockchain runtime from scratch
- 🔄 CI/CD pipelines covering smart contracts → backend APIs → E2E dashboard automation
